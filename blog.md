## 这是什么

TeslaMate MCP 是一个让 **AI 助手读懂你特斯拉** 的工具。

接入 AI 助手（Claude Code、OpenClaw）后，你说什么它就查什么，不用学任何技术，不用记任何命令。

**特点：**
- 🔒 安全 — 只有读取，没有任何车辆控制
- 🚫 不需要 Tesla 账号 — 直接从本地数据库读
- 📊 30 个功能 — 覆盖你关心的方方面面
- 🎯 有意思 — 有成就系统、盲盒目的地、月度报告

## 功能清单

### 🚗 车辆状态

| 功能 | 能查什么 |
|------|----------|
| **当前状态** | 电量还剩多少、续航多少公里、车在哪儿、空调开没开 |
| **实时状态** | GPS、电池温度、胎压、是否在充电 |
| **胎压监测** | 四个轮子各多少胎压，有没有异常 |
| **胎压历史** | 最近胎压变化记录 |

### 📊 行程记录

| 功能 | 能查什么 |
|------|----------|
| **行程列表** | 最近去了哪儿、跑了多远、用了多少电 |
| **驾驶评分** | 加速习惯、刹车习惯、速度习惯，各项打分 |
| **按类型筛选** | 通勤多少次、购物多少次、长途多少次 |
| **最长行程** | 你的里程纪录是去哪儿的 |
| **常去地点** | 你最常跑的几个地方是哪儿 |
| **地点历史** | 在各个地点各停留了多久 |

### 🔋 电池情况

| 功能 | 能查什么 |
|------|----------|
| **充电记录** | 每次充电的时间、地点、充了多少 |
| **地点充电习惯** | 在家充得多还是在外面充得多 |
| **电池健康度** | 电池衰减趋势，看看这几个月掉没掉 |
| **驻车掉电** | 过夜停着能掉多少电（传说中的吸血鬼耗电） |

### ⚡ 能耗分析

| 功能 | 能查什么 |
|------|----------|
| **能耗趋势** | 最近几周能耗是上升还是下降 |
| **温度影响** | 天冷了能耗涨了多少，天热了呢 |
| **月度报告** | 这个月跑了多少、充了多少、能耗多少、和上个月对比 |
| **月度汇总** | 表格形式，里程/kWh/费用/能耗一目了然 |

### 💰 省钱 & 环保

| 功能 | 能查什么 |
|------|----------|
| **省钱 scorecard** | 买了电车之后比油车省了多少钱 |
| **行程费用估算** | 去某个地方大概要花多少电费 |
| **eco 对比** | 比起油车，你省了多少钱、减排了多少 CO₂、相当于种了几棵树 |

### 🏆 好玩的 ⭐

这部分是**最有意思**的：

| 功能 | 能查什么 |
|------|----------|
| **隐藏成就** | 解锁一些有趣的驾驶成就（见下文） |
| **旅行叙事** | 某次行程生成一个故事版，可以当 Vlog 脚本 |
| **周末盲盒** | 你曾经心血来潮去过、但再也没去过的神秘地点是什么？ |
| **月度报告（精美版）** | 自动生成一份带 Emoji 的 Markdown 月报 |
| **车辆人设** | 你的车是"活跃派"还是"省电达人"还是"夜猫子"？ |
| **充电详情** | 某次充电的完整物理参数：电压、电流、功率变化 |

## 好玩的详细说说

### 🎖️ 隐藏成就系统

系统会自动检测你的驾驶数据，解锁各种有意思的称号：

| 成就 | 什么条件 |
|------|----------|
| 🏅 **极限续航幸存者** | 续航只剩 50km 以下还继续开 |
| 🌙 **午夜幽灵** | 凌晨 2-4 点还在开车 |
| ❄️ **冰雪勇士** | 在 0°C 以下的天气驾驶 |
| 🔋 **充电狂人** | 一个月充电超过 20 次 |
| 🏃 **通勤达人** | 单月通勤超过 1000 公里 |
| 🌡️ **温差战士** | 24 小时内经历超过 20°C 温差 |

### 🎲 周末盲盒

你有没有过那种"一时兴起跑了个巨远的地方然后再也没去过"的经历？

这个功能会从你的历史轨迹里，找出那些你去过 **但只去过一次** 的地点，然后推荐给你：

```
🎲 你的周末盲盒：
「崇礼滑雪场」

你曾在 2024年1月14日去过这里，单程 187km
那天能耗 18.2 Wh/km（天冷正常）
充电花了 ¥19.4

要不要再来一次说走就走的旅行？🚗
```

### ✈️ 旅行叙事生成

某次出行后，让 AI 给你讲个故事：

```
📍 3月15日 · 北京 → 秦皇岛

早上8点32分，满电出发（98%，489km续航）
高速上开着 Autopilot，平均时速 110km
9点47分到达，电量还剩 62%（298km续航）
全程 287km，耗时 1小时15分钟
能耗 15.8 Wh/km，高速表现不错

充电：在酒店充了 32.4 kWh，花了 ¥19.4
晚上6点充完，8点吃完海鲜回北京
```

这段内容可以直接当 Vlog 脚本用。

### 📋 月度报告

每月初自动生成上个月的驾驶报告：

```
# 🚗 2024年3月驾驶报告

## 概览
- 行驶里程：1,247 km
- 充电次数：8 次
- 总耗电：186.5 kWh
- 平均能耗：14.2 Wh/km（比上月 ↓ 3%）

## 💰 省钱计算
- vs 燃油车节省：¥ 486
- CO₂ 减排：238 kg
- 相当于种了：12 棵树 🌳

## 🏆 本月成就

- 最高能耗：22.1 Wh/km（暴风雪那天）
- 最低能耗：11.8 Wh/km（高速巡航）
- 解锁成就：❄️ 冰雪勇士

## 📍 常去地点
1. 家 — 停留 312 小时
2. 公司 — 停留 89 小时
3. 望京 SOHO — 停留 12 小时
```

## 怎么用

### 第一步：部署服务

在群晖 NAS 的 Docker 里运行（和 TeslaMate 共用同一个 docker-compose.yml），你可以自己编译Docker镜像，也可以使用我编译好的代码（可能随时会更新、可能会有Bug😄）：

```yaml
services:
  teslamate-mcp:
    image: ghcr.io/6547709/teslamate-mcp:latest
    container_name: teslamate-mcp
    restart: always
    ports:
      - "3002:8080"
    environment:
      - TESLAMATE_DB_HOST=database
      - TESLAMATE_DB_PORT=5432
      - TESLAMATE_DB_USER=teslamate
      - TESLAMATE_DB_PASS=你的密码
      - TESLAMATE_DB_NAME=teslamate
      - USE_METRIC_UNITS=true      # 公制单位（公里）
      - TESLA_ELECTRICITY_RATE_RMB=0.6  # 电费价格
      - TESLA_CAR_ID=1             # 车辆ID
      - TESLA_BATTERY_KWH=75       # 电池容量
      - TESLA_BATTERY_RANGE_KM=525 # 满电续航
    depends_on:
      - database
```

启动后服务就运行起来了。

### 第二步：连接 AI 助手

在 Openclaw中直接对话添加，“帮我添加一个teslamate-mcp，地址为http://<你的NAS地址>:3002/mcp”；

在 Claude Code 的设置文件（`~/.claude/settings.json` 或项目里的 `.mcp.json`）添加：

```json
{
  "mcpServers": {
    "teslamate": {
      "url": "http://你的NAS地址:3002/mcp"
    }
  }
}
```


### 第三步：开始对话

打开 Openclaw/Claude Code，想问什么就说什么：

```
"帮我看看上个月的驾驶报告"

"我买电车之后比油车省了多少钱"

"解锁了哪些成就"

"生成一个周末盲盒"

"我这车电池健康度怎么样了"
```

## Q&A

**Q: 这个会控制我的车吗？**
A: 完全不会。只有读取，没有任何命令下发。

**Q: 需要 Tesla 官方账号吗？**
A: 不需要。数据直接从本地 TeslaMate 数据库读取。

**Q: 我的数据安全吗？**
A: 数据都在你自己的Teslamate环境中，不经过任何第三方服务器。

**Q: 支持多辆车吗？**
A: 目前默认查询第一辆车，可以在配置里改车辆 ID。

**Q: 提供源代码吗？**
A: 是的，源代码已经在附件中了，源代码中包含详细的说明和指导，实现代码只有一个tesla.py代码文件，非常容易扩展能力。
